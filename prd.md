# PRD: Multi-Capability AI Agent Platform

## 1. Project Overview

A web-based platform where administrators can configure specialized AI Agents with distinct personas, knowledge bases, and functional capabilities. Users can interact with these agents through a chat interface, receiving responses that range from simple text to generated images and downloadable PDF reports.

---

## 2. Target Audience

* **Admins:** Technical or semi-technical users who configure agent behavior and upload proprietary data.
* **End Users:** Individuals seeking specific automated tasks (e.g., a "Legal Assistant" agent or a "Creative Designer" agent).

---

## 3. Functional Requirements

### 3.1 Admin Dashboard (Agent CRUD)

The admin needs a "Control Center" to manage the lifecycle of AI agents.

* **Persona Configuration:** Define Name, Avatar, System Prompt (Instructions), and Temperature.
* **Model Selection:** Integration with **OpenRouter.ai** to select specific LLMs (e.g., Claude 3.5 Sonnet for logic, GPT-4o for general tasks, or Llama 3 for speed).
* **Knowledge Base Management:**
* Upload PDF, TXT, and Docx files.
* *Technical Note:* Files must be "chunked" and stored in a Vector Database (like Pinecone or a local PGVector) to enable RAG (Retrieval-Augmented Generation).


* **Capability Toggles:**
* **Text Only:** Standard conversational mode.
* **Image Generation:** Enable tools to trigger image APIs.
* **PDF Export:** Enable the agent to populate a Laravel Blade template with chat data and export it as a PDF.



### 3.2 User Chat Interface

* **Agent Marketplace/List:** A gallery view for users to select the agent they want to talk to.
* **Threaded Conversations:** Standard chat UI with markdown support.
* **Capability-Specific Outputs:**
* **Images:** Rendered directly in the chat bubble.
* **PDFs:** A "Download Report" button generated by the agent upon task completion.

## 4. Technical Architecture (Laravel Fullstack)

| Component | Technology |
| --- | --- |
| **Backend** | Laravel 11+ |
| **Frontend** | Livewire (for real-time chat) or Inertia.js (Vue/React) |
| **LLM Gateway** | [OpenRouter.ai](https://openrouter.ai/) API |
| **PDF Generation** | `barryvdh/laravel-dompdf` or `spatie/browsershot` |
| **Image Gen** | DALL-E 3 API (via OpenRouter or OpenAI directly) |
| **Vector Search** | `openai-php/client` with a Vector Store (for file-based knowledge) |
| **Database** | PostgreSQL (recommended for PGVector support) |

---

## 5. Key Workflows

### A. The "Knowledge" Workflow

1. Admin uploads `company_policy.pdf`.
2. Laravel Job parses the text and sends embeddings to a vector database.
3. User asks: "What is the vacation policy?"
4. System searches the vector DB, finds the relevant text, and sends it to OpenRouter as context.

### B. The "PDF Template" Workflow

1. User asks the "Analyst Agent" to create a summary.
2. The agent generates a JSON structure of the summary.
3. Laravel receives the JSON, passes it to a **Blade Template**, and generates a PDF.
4. The user receives a download link in the chat.

---

## 6. Data Schema (Simplified)

* **`agents`**: id, name, model_id (OpenRouter), system_prompt, capabilities (JSON), avatar.
* **`knowledge_sources`**: id, agent_id, file_path, status (processed/pending).
* **`conversations`**: id, user_id, agent_id.
* **`messages`**: id, conversation_id, role (user/assistant), content, metadata (for image URLs or PDF paths).

---

## 7. Success Metrics

* **Latency:** OpenRouter response time should stay under 3 seconds for text.
* **Accuracy:** RAG retrieval should provide relevant context from uploaded files 90% of the time.
* **Conversion:** Successful generation of PDFs/Images based on user prompts.
